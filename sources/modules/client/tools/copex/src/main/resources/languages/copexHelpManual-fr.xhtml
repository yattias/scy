SCYExperimentalDesign est un outil pour vous aider à construire un protocole expérimental, afin de décrire votre expérience.

Cette fenêtre d'aide vous donne des astuces et des conseils pour créer votre protocole. Selon la mission dans laquelle vous êtes impliqué, SCYExperimentalDesign peut être configurer différemment. Ainsi toutes les fonctionnalités décrites ci-dessous ne sont peut-être pas pertinentes dans votre cas...

D'abord nous allons décrire la façon dont est structuré un protocole dans SCYExperimentalDesign :

Un protocole fait référence à une question principale ou Ã  un objectif. Si la question n'est pas renseignée, vous ne serez pas en mesure d'aller plus loin dans la description de votre protocole. Alors remplir ce champ est la première tâche à faire !
Vous pouvez écrire des hypothèses à côté de l'icône de l'ampoule. Les hypothèses sont utlisées pour décrire la réponse possible à la question et/ou décrire certaines propriétés que vous croyez être vraies dans votre protocole et qui ne seront pas testés.
Le principe général consiste en la description approximative du protocole. Il peut être intéressant ici d'écrire les grandeurs que vous voulez mesurer pour tester votre hypothèse. Eventuellement le principe peut également être décrit par un schéma.
Une fois écrit la question, vos hypothèses et le principe général du protocole, vous avez une meilleure idée du protocole. Il est temps d'indiquer quel matériel vous allez utliser. En fonction de la configuration de SCYExperimentalDesign, vous avez différentes options afin de choisir le matériel (éventuellement vous pouvez ajouter de nouveaux matériels).
Le protocole en lui-même sera décrit dans une de ces deux icônes (en fonction de la configuration de SCYExperimentalDesign). En dessous de cette icône principale, vous devrez ajouter des tâches et sous-tâches qui décriront chaque action à effectuer pendant l'expérience.
Cette icône représente une tâche. Chaque tâche peut encore être décomposée en sous-tâches.
Si un petit verrou est présent sur une tâche, cela signifie que vous n'avez pas le droit de faire tout ce que vous voulez (par exemple la déplacer ou la modifier).
Si un petit avertissement est présent sur une tâche, cela signifie que cette tâche est mal configurée, vous devez la corriger.
Ce sont des tâches spéciales de votre protocole, appelées étapes. Ces étapes vous aident à structurer votre protocole et elles doivent contenir des actions.
Les icônes violettes représentent les actions de votre protocole. Elles doivent décrire l'activité de la personne qui va exécuter votre protocole. La description des actions (matériels, paramètres) doit être aussi précise que possible.
Dans certains cas, SCYExperimentalDesign va collecter toutes les données que vous avez l'intention de mesurer au cours de votre expérience. Ces données seront rassemblées dans cette section, appelée "la feuille de données", afin de vous aider pendant votre expérimentation.
Cette section sera utilisée pour évaluer votre protocole. Une fois de plus il peut y avoir différentes configurations. Vous pouvez ajouter vos commentaires personnels dans une zone de texte ou vous pouvez être aidé par des lignes directrices. Parfois, certaines observations peuvent être faites par un tuteur automatique.

Vous pouvez modifier votre protocole en cliquant sur chaque item. Regardons les principales fonctionnalités des boutons de la barre de menu :

Affiche ou masque l'espace des hypothèses.
Affiche ou masque l'espace du principe général.
Affiche ou masque l'espace de l'évaluation.
Affiche ou masque vos commentaires.
Développe la visualisation du protocole au niveau 1, 2, 3...
Exporte votre protocole au format PDF. Utilisez cette fonctionnalité pour imprimer votre protocole.

Maintenant que SCYExperimentalDesign a été décrit, c'est à votre tour de concevoir vos expérimentations.
